# Fortune Compass UI設計方針

## 1. 全体コンセプト

**テーマ: 「宇宙の神秘」**

神秘的でありながら、読みやすく使いやすいUI。ダークテーマをベースに、紫と金のアクセントで占いの世界観を演出する。

### デザイン原則
1. **明瞭性** — 占い結果が一目で分かる。スコアは数値 + 星で視覚化
2. **没入感** — ダークテーマ + グラデーションで神秘的な雰囲気を演出
3. **シンプルさ** — 1画面1目的。迷わない導線設計
4. **モバイルファースト** — スマホでの操作を最優先に設計

**要素:**

- 名前: 1文字以上
- フリガナ: カタカナのみ
- 生年月日: 必須、未来の日付は不可
- 血液型: 未選択でもOK（血液型占いは使えない旨を表示）

### 2.8 追加占術の結果ページ

全ての追加占術結果ページは共通レイアウトパターンに従う: ヒーローセクション（アイコン + タイトル）、結果カード群、OtherFortunes + ShareButtons。以下に各占術固有のUI要素を記載する。

**ヒーロー:** 干支アイコン + 干支名（例: 午年）
**結果カード:**
1. 今日の運勢スコア（★表示）
2. 性格特徴カード
3. 相性の良い干支
4. アドバイス

#### 2.8.3 動物占い結果（`/fortune/animal`）

**ヒーロー:** 動物キャラクターアイコン + キャラ名（例: 「全力疾走するチーター」）
**結果カード:**
1. 性格・行動パターン
2. 対人関係の傾向
3. 相性の良い動物キャラ
4. アドバイス

#### 2.8.4 誕生花占い結果（`/fortune/birth-flower`）

**結果カード:**

#### 2.8.5 誕生石占い結果（`/fortune/birthstone`）

**結果カード:**
1. 石の持つパワー・効果
2. 性格との関連
3. アドバイス

#### 2.8.6 四柱推命結果（`/fortune/shichuu`）

**ヒーロー:** 四柱推命アイコン + タイトル
**結果カード:**
1. **三柱表示**（年柱・月柱・日柱の天干地支を表形式で表示）
2. 通変星・十二運カード
3. 性格・運勢の解説
4. アドバイス

**固有UI:** 三柱を横並びの表（テーブル形式）で表示。各柱に天干・地支を上下に配置。

#### 2.8.7 曜日占い結果（`/fortune/weekday`）

**ヒーロー:** カレンダーアイコン + 生まれた曜日（例: 「金曜日生まれ」）
**結果カード:**
1. 性格特徴
2. 今日の運勢
3. アドバイス

#### 2.8.8 風水占い結果（`/fortune/fengshui`）

**ヒーロー:** 風水アイコン + 本命卦名
**結果カード:**
1. **方位コンパス**（SVGまたはCSS描画の八方位図。吉方位を緑、凶方位を赤で色分け表示）
2. 吉方位・凶方位の詳細リスト
3. ラッキーカラー・ラッキーアイテム
4. アドバイス

**固有UI:** 八角形の方位コンパスで吉凶を視覚的に表現。

#### 2.8.9 おみくじ結果（`/fortune/omikuji`）

**ヒーロー:** おみくじアイコン + タイトル
**結果カード:**
1. **運勢ランク表示**（大吉〜凶を大きな文字 + 背景色で表示。大吉=金、吉=紫、中吉=青、小吉=緑、末吉=黄、凶=灰、大凶=赤）
2. 各項目（願望・待人・失物・旅行・商売・学問・恋愛など）
3. 総合メッセージ

**固有UI:** ランク表示を画面中央に大きく表示。ランクに応じた背景色グラデーション。

#### 2.8.10 ルーン占い結果（`/fortune/rune`）

**ヒーロー:** ルーンアイコン + タイトル
**結果カード:**
1. **3つのルーンストーン表示**（過去・現在・未来の3石を横並び。各石にルーン文字 + 名前 + 正逆を表示）
2. 各ルーンの詳細解説
3. 総合メッセージ

**固有UI:** タロットと同様の3枚横並びレイアウト。各石は円形背景 + ルーン文字を中央に配置。

#### 2.8.11 夢占い結果（`/fortune/dream`）

**ヒーロー:** 夢占いアイコン + タイトル
**入力UI:**
- **キーワード入力フィールド**（テキスト入力。夢の中で印象的だったキーワードを入力。例: 「海」「空を飛ぶ」「猫」）
- 送信ボタン

**結果カード:**
1. 入力キーワードの象徴・意味
2. 深層心理の解説
3. 運勢へのアドバイス

**固有UI:** 他の占術と異なり、プロフィール情報だけでなくキーワード入力が必要。入力欄 + 結果表示の2段階構成。

#### 2.8.12 手相占い結果（`/fortune/palm`）

**ヒーロー:** 手相アイコン + タイトル
**入力UI:**
- **画像アップロード**（手のひらの写真をアップロード。カメラ撮影 or ギャラリーから選択）
- プレビュー表示 + 送信ボタン

**結果カード:**
1. 検出された手相線の解説（生命線・頭脳線・感情線・運命線など）

## 7. 追加UI要素

### スキップナビゲーション
- ページ最上部に「メインコンテンツへスキップ」リンク（sr-only、フォーカスで表示）

### 言語切替（LanguageSwitcher）
- ヘッダー右側に Globe アイコン + "EN"/"JP" テキスト
- クリックで日本語/英語を切替（localStorage に保存）

### ローディング状態（更新）
- `role="status"` + `aria-live="polite"` でアクセシビリティ対応
- 「占い一覧に戻る」リンク付き

### アクセシビリティ対応
- 全インタラクティブ要素に `focus:ring-2 focus:ring-mystic-purple/60`
- フォーム要素に `aria-required`, `aria-invalid`, `aria-describedby`
- 血液型ボタンに `aria-pressed`
- スコア表示に `role="img"` + `aria-label="スコア: N点（5点満点）"`
- 装飾アイコンに `aria-hidden="true"`
- `<section>`, `<nav>`, `<fieldset>/<legend>` によるセマンティック構造

### OGP / SNS メタデータ
- Open Graph: title, description, url, siteName, locale, type, images
- Twitter Card: summary_large_image
- 動的OG画像: `opengraph-image.tsx`（Next.js ImageResponse API）

### PWA
- `manifest.json`: アプリ名、アイコン(192/512)、テーマカラー、表示モード
- SVG アイコン3種（通常192/512、マスカブル）
